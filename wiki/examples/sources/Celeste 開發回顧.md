# Celeste 開發回顧

## Summary

[[wiki/examples/entities/Maddy Thorson]] 在這篇文章中回顧了 Celeste 從 2016 年 Game Jam 原型到 2018 年正式發售的開發歷程。遊戲以 [[wiki/examples/concepts/像素藝術]] 風格呈現，由兩人核心團隊完成。文章強調了「快速原型 → 迭代手感 → 後期擴展」的開發方法論。

## Key Claims

- Celeste 原型在 PICO-8 上用 4 天完成，核心跳躍和衝刺手感在 jam 期間已經「對了」
- 選擇像素藝術不是懷舊，而是務實——兩個人不可能做 3D 或高解析度 2D
- 「Assist Mode」（輔助模式）讓所有玩家都能通關，沒有犧牲核心難度
- 難度曲線的祕訣：每個房間只考驗一個技巧，死了立刻重來，沒有懲罰

> [!key-insight] 核心洞見
> 像素遊戲的美術限制反而成為設計優勢——更少的像素迫使團隊把精力集中在手感和關卡設計上。

## Entities Mentioned

- [[wiki/examples/entities/Maddy Thorson]] — 遊戲設計師兼程式設計師，Celeste 的核心創作者

## Concepts Introduced

- [[wiki/examples/concepts/像素藝術]] — Celeste 選擇的視覺風格及其對開發效率的影響
- Assist Mode — 無障礙設計的業界標竿案例
- Juice（遊戲手感）— 畫面震動、粒子效果、音效回饋的總稱

## Notes

這篇文章是了解獨立像素遊戲開發流程的經典案例。Celeste 後來獲得 TGA 2018 最佳獨立遊戲，證明像素藝術在商業上仍然可行。
